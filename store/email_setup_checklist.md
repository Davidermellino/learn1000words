# Branded Auth Email Setup — learn1000words (Resend + Supabase)

Goal: send Supabase auth emails from a branded address on **learn1000words.com**
instead of Supabase's default sender.

- **Sending domain:** `learn1000words.com`
- **Auth sender (outbound only):** `noreply@learn1000words.com`
- **Reply-to / contact:** `ermellinodavide@gmail.com`
- **Only auth email in use:** signup confirmation. (Password reset and magic
  link are not wired in the app, so there is nothing to configure for them yet.)

> ⚠️ Every value marked **(copy from dashboard)** is generated by Resend or
> Supabase when you perform the step. Do **not** invent DKIM keys, SPF strings,
> or API keys — copy the exact value the dashboard shows you.

The repo-side pieces are already done: the confirmation template lives at
`store/email_templates/confirm-signup.html`, and the contact address is filled
into `store/privacy_policy.md`, `docs/privacy-policy.html`, and
`docs/delete-account.html`.

---

## Do these steps in order

### 1. Create a Resend account
1. Sign up at <https://resend.com> (free tier is fine to start).

### 2. Add the domain in Resend
1. In Resend, go to **Domains → Add Domain**.
2. Enter `learn1000words.com` and submit.
3. Resend now shows a list of DNS records to add. Leave this page open — you
   need its exact values for step 3. It will include:
   - A **DKIM** record (type `TXT` or `CNAME`, host like `resend._domainkey`) — **(copy from dashboard)**.
   - An **SPF** record (type `TXT`, value contains `include:...resend...` / an MX for the bounce subdomain) — **(copy from dashboard)**.
   - Possibly an **MX** record on a `send.` (or similar) subdomain for return-path/bounces — **(copy from dashboard)**.

### 3. Add the DNS records at your domain registrar
1. Log in to wherever `learn1000words.com` is registered (the registrar/DNS host).
2. Open the DNS management page for the domain.
3. For **each** record Resend listed in step 2, create a matching record:
   - Set **Type**, **Host/Name**, and **Value** exactly as shown in Resend — **(copy from dashboard)**.
   - If the registrar auto-appends the domain to the host, enter only the
     subdomain part (e.g. `resend._domainkey`, not the full FQDN).
   - Leave TTL at the default (or `3600`).
4. Save. DNS propagation can take from minutes up to ~48h (usually fast).

### 4. Verify the domain in Resend
1. Back in Resend's **Domains** page, click **Verify** for `learn1000words.com`.
2. Wait until every record shows **Verified / green**. Re-check after a while if
   some are still pending — do not proceed until the domain is fully verified.

### 5. Create the Resend API key
1. In Resend, go to **API Keys → Create API Key**.
2. Name it e.g. `supabase-auth-smtp`, permission **Sending access**.
3. Copy the key now — **(copy from dashboard)**. Resend shows it only once.
   - Store it in a password manager. **Never commit it to this repo.**

### 6. Configure custom SMTP in Supabase
1. Open the Supabase dashboard for this project.
2. Go to **Project Settings → Authentication → SMTP Settings**
   (in newer UIs: **Authentication → Emails → SMTP Settings**).
3. Enable **"Use a custom SMTP server"** and fill in:
   - **Host:** `smtp.resend.com`
   - **Port:** `465` (SSL) — or `587` (STARTTLS) if 465 is blocked.
   - **Username:** `resend`
   - **Password:** the Resend API key from step 5 — **(paste from step 5)**.
   - **Sender email:** `noreply@learn1000words.com`
   - **Sender name:** `learn1000words`
   - **Reply-to** (if the field exists): `ermellinodavide@gmail.com`
4. Save.

### 7. Install the branded confirmation template in Supabase
1. Go to **Authentication → Emails → Templates → "Confirm signup"**.
2. Open `store/email_templates/confirm-signup.html` from this repo, copy the
   `<body>` markup (everything below the comment block), and paste it into the
   template's HTML/message field.
   - Keep the `{{ .ConfirmationURL }}` placeholder intact — Supabase fills it in.
3. Optionally set the **Subject** to: `Confirm your learn1000words email`.
4. Save.

### 8. Send a real test
1. In the app, register a brand-new account with an email address you control.
2. Confirm the email arrives:
   - **From:** `learn1000words <noreply@learn1000words.com>`
   - Branded template renders, and the **Confirm my email** button works.
   - Reply goes to `ermellinodavide@gmail.com` (if reply-to was set).
3. If it lands in spam or fails, re-check that all DNS records in step 3 are
   verified (step 4) and that the SMTP credentials in step 6 are correct.
